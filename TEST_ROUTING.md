# การทดสอบ Routing

## วิธีทดสอบ

### 1. ทดสอบเอกสาร Incentive (ควรไปที่ DocumentViewer)
1. เปิด browser ไปที่ http://localhost:3000
2. ค้นหา "incentive"
3. คลิกปุ่ม "ดูเอกสาร" หรือคลิกที่ชื่อเอกสาร
4. ตรวจสอบ URL ควรเป็น `/document/inc_001` หรือ `/document/inc_002`
5. ควรเห็นหน้า DocumentViewer ที่แสดง PDF

### 2. ทดสอบเอกสารลา (ควรไปที่ LeavePage)
1. เปิด browser ไปที่ http://localhost:3000
2. ค้นหา "ลา" หรือ "พักร้อน"
3. คลิกปุ่ม "ดูเอกสาร" หรือคลิกที่ชื่อเอกสาร
4. ตรวจสอบ URL ควรเป็น `/leave/hr_003`, `/leave/hr_004`, หรือ `/leave/hr_005`
5. ควรเห็นหน้า LeavePage ที่มี UI เฉพาะสำหรับการลา

## Logic การตรวจสอบ

ฟังก์ชัน `isLeaveDocument()` จะตรวจสอบ:
- คำสำคัญในชื่อ: 'ลา', 'พักร้อน', 'ป่วย', 'กิจส่วนตัว', 'คลอด', 'leave'
- คำสำคัญในเนื้อหา: เช่นเดียวกับข้างต้น
- หมวดหมู่ 'ทรัพยากรบุคคล' ที่มีคำว่า 'ลา' ในชื่อหรือเนื้อหา

## ตัวอย่างเอกสาร

### เอกสารที่ควรไปหน้าลา:
- hr_003: นโยบายการลาพักร้อนประจำปี 2024
- hr_004: คู่มือการขอลาป่วยและลากิจ
- hr_005: นโยบายลาคลอดและลาเลี้ยงดูบุตร

### เอกสารที่ควรไป DocumentViewer:
- inc_001: Incentive ระดับสาขา 2024
- inc_002: Incentive ระดับผู้จัดการพื้นที่และภาค 2024
- kpi_001: ตัวชี้วัดผลงาน KPI
- และเอกสารอื่นๆ ที่ไม่เกี่ยวกับการลา

## หากพบปัญหา

1. เปิด Developer Console (F12)
2. ดูที่ Console tab
3. ตรวจสอบ error messages
4. ตรวจสอบ Network tab เพื่อดู requests
5. ตรวจสอบว่า URL ที่ redirect ไปถูกต้องหรือไม่
